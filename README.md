## Promises/A+ Implementations (ES6/ES2015 compatible)

### Strict Implementations
These implement no more or less than the es6 spec. They make great polyfills and are exceptionally compatible with native promises.

* <b><code>&nbsp;&nbsp;&nbsp;137⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;17🍴</code></b> [pinkie](https://github.com/floatdrop/pinkie)) - Ponyfill. Node-oriented, but <b><code>&nbsp;14723⭐</code></b> <b><code>&nbsp;&nbsp;1201🍴</code></b> [browserifyable](https://github.com/substack/node-browserify)). *Extremely* small implementation.
* <b><code>&nbsp;&nbsp;&nbsp;717⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;69🍴</code></b> [native-promise-only](https://github.com/getify/native-promise-only)) - Polyfill. Browser and node-compatible.
* <b><code>&nbsp;&nbsp;7287⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;591🍴</code></b> [es6-promise](https://github.com/stefanpenner/es6-promise)) - Opt-in polyfill. A strict-spec subset of rsvp.js.
* <b><code>&nbsp;&nbsp;&nbsp;740⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;60🍴</code></b> [lie](https://github.com/calvinmetcalf/lie)) - Small, browserifyable with an opt-in polyfill.

### Implementations with extras
All of these provide more features than the language yet remain compatible. Node + Browsers for all.

* <b><code>&nbsp;20735⭐</code></b> <b><code>&nbsp;&nbsp;2333🍴</code></b> [bluebird](https://github.com/petkaantonov/bluebird)) - Fully featured, extremely performant. Long stack traces & generator/coroutine support.
* <b><code>&nbsp;&nbsp;&nbsp;277⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;20🍴</code></b> [creed](https://github.com/briancavalier/creed)) - Hyper performant & full featured like Bluebird, but FP-oriented. Coroutines, generators, promises, ES2015 iterables, & fantasy-land spec.
* <b><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;?⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;?🍴</code></b> [rsvp.js](https://github.com/tildeio/rsvp.js/)) - Lightweight with a few extras. Compatible down to IE6!
* <b><code>&nbsp;15208⭐</code></b> <b><code>&nbsp;&nbsp;1188🍴</code></b> [Q](https://github.com/kriskowal/q)) - One of the original implementations. Long stack traces and other goodies.
* <b><code>&nbsp;&nbsp;2594⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;315🍴</code></b> [then/promise](https://github.com/then/promise)) - Small with `nodeify`, `denodify` and `done()` additions.
* <b><code>&nbsp;&nbsp;3434⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;393🍴</code></b> [when.js](https://github.com/cujojs/when)) - Packed with control flow, functional, and utility methods.


### Fallbacks
* 🌎 [native-or-bluebird](www.npmjs.com/package/native-or-bluebird) - Helps transition to completely native.
* <b><code>&nbsp;&nbsp;&nbsp;116⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;17🍴</code></b> [pinkie-promise](https://github.com/floatdrop/pinkie-promise)) - Use native, or fall back to `pinkie`. Great for node library authors.
* <b><code>&nbsp;&nbsp;&nbsp;181⭐</code></b> <b><code>&nbsp;&nbsp;&nbsp;&nbsp;16🍴</code></b> [any-promise](https://github.com/kevinbeaty/any-promise)) - Loads the first available implementation. Safe for browserify.

## Convenience Utilities
Native and strictly spec-compliant promises are awesome for compatibility, future-proofness, library authors, and browsers. However, libraries like bluebird patch goodies onto the `Promise` constructor and prototype. Solution? tiny modules of course!
